ParallelGit is an open source project built upon JGit.

It provides you with easy access to your local Git repository.


Yet another data storage

Every non-trivial project needs some sort of a data storage.

Many projects use a SQL database. Some are better off with a NoSQL database. The dodgy ones simply drop a few files in your home directory.

A Git repository is just another place to store your data.


The keeper of the history

We all know that Git is awesome at storing text files that change over time.
If you are managing some text files and want a system to keep a close eye on them, Git is a good candidate for this job.

In fact, Git doesn't care about the format of a file.
You can drop a binary file into a Git repository and Git will keep a nice and neat history of this file.

What Git really offers is the ability to keep the entire history of a data structure at a very low cost.


A time machine

People use Git know how to change the state of a repository by checking out a commit.

It is like you have a time machine that allows you to jump from one point to another on the time line.

If you want your Java program to have such power, you should consider adding a library that can talk to a Git repository such as JGit.


A touch screen for your time machine

Let me just tell you that using the JGit API is not the most enjoyable thing in the world.

Imagine you have very powerful a time machine.
It can do 2000 things. (10 locations * 200 times)
There are 10 buttons in front of you.
The machine will take you to the place you want to go, if you press the buttons in a certain order.
The machine will either blow up or take you to a random place, if you get the combination wrong.

Are you comfortable with this time machine?

How would you like a touch screen for this time machine?
How would you like having all the options displayed right in front your eyes. Anything you want to do is just a few taps away.
Sounds good?
ParallelGit may just be the touch screen for your time machine.
